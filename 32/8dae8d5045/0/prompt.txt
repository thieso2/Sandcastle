# Plan: Replace WeTTY with ttyd (tmux + shell terminals)

## Context
WeTTY requires SSH keypair generation, key injection via `docker exec`, and a sidecar container per terminal session. Replacing it with ttyd running **inside** the sandbox container eliminates all of this complexity. Two ttyd instances run permanently on fixed ports inside each sandbox; Traefik routes directly to them. No sidecars, no keypairs, no background jobs.

## Key architectural facts
- Sandbox containers use `"NetworkMode" => "sandcastle-web"` (SandboxManager:4,65) — they're already on the same Docker network as Traefik and Rails
- ttyd running inside the sandbox is reachable as `http://{sandbox.full_name}:768x` from Traefik without any host port mapping
- ttyd exposes HTTP + WebSocket on the same port (Traefik proxies both transparently, same as WeTTY on port 3000)
- `tmux` is already installed in the sandbox image (Dockerfile:9)

## Ports
- Port **7681** → `tmux new-session -A -s main` (unlimited clients, re-attachable)
- Port **7682** → `bash -l` (plain login shell, one client at a time)

## Files to change

### 1. `images/sandbox/Dockerfile`
Add ttyd binary after existing tools:
```dockerfile
# ttyd - web terminal
RUN ARCH=$(dpkg --print-architecture) \
    && TTYD_ARCH=$([ "$ARCH" = "arm64" ] && echo "aarch64" || echo "x86_64") \
    && curl -fsSL "https://github.com/tsl0922/ttyd/releases/latest/download/ttyd.${TTYD_ARCH}" \
       -o /usr/local/bin/ttyd \
    && chmod +x /usr/local/bin/ttyd
```
Update `EXPOSE 22 5900 6080` → `EXPOSE 22 5900 6080 7681 7682`

### 2. `images/sandbox/entrypoint.sh`
Add before `exec /usr/sbin/sshd -D -e` (line 107):
```bash
# Start ttyd web terminals (run as sandbox user)
# Port 7681: tmux session (persistent, re-attaches; unlimited clients)
# Port 7682: plain login shell (one client at a time)
su -s /bin/bash "$USERNAME" -c \
  "ttyd -W -m 0 -p 7681 tmux new-session -A -s main &>/var/log/ttyd-tmux.log &"
su -s /bin/bash "$USERNAME" -c \
  "ttyd -W -m 1 -p 7682 bash -l &>/var/log/ttyd-shell.log &"
```

### 3. `app/services/terminal_manager.rb` — complete rewrite
Remove: all WeTTY/sidecar code, SSH keypair generation, `connect_sandbox_to_network`, `copy_key_to_container`, `remove_pubkey`, `cleanup_orphaned` (replaced).

New shape:
- `TMUX_PORT = 7681`, `SHELL_PORT = 7682`
- `open(sandbox:, type: "tmux")` — writes Traefik config (idempotent), returns terminal URL
- `active?(sandbox:, type: "tmux")` — TCP probe to `sandbox.full_name:port`
- `close(sandbox:)` — deletes Traefik config
- `cleanup_orphaned` — scans `terminal-*.yml` files, deletes any whose sandbox is gone/not running

Traefik config writes **two routers + two services** in one file (`terminal-{id}.yml`), sharing one `forwardAuth` middleware:
- router `terminal-{id}-tmux`: `PathPrefix(/terminal/{id}/tmux)` → service pointing to port 7681
- router `terminal-{id}-shell`: `PathPrefix(/terminal/{id}/shell)` → service pointing to port 7682
