# Plan: TCP Port-Forward Routes

## Context

Routes currently support only HTTPS (layer 7, domain-based via Traefik `Host()` rules on port 443).
The new "tcp" mode adds layer-4 port forwarding: Traefik listens on a public port (e.g. 3000)
and forwards raw TCP to the container port. This is useful for databases, raw TCP services, or
anything that isn't HTTP. Port range: 3000–3099 (100 slots), pre-declared in the Traefik static
config and opened in UFW by the installer.

## RouteManager (`app/services/route_manager.rb`)

**Constants**
```ruby
TCP_PORT_MIN = ENV.fetch("SANDCASTLE_TCP_PORT_MIN", 3000).to_i
TCP_PORT_MAX = ENV.fetch("SANDCASTLE_TCP_PORT_MAX", 3099).to_i
TCP_PORT_RANGE = (TCP_PORT_MIN..TCP_PORT_MAX)
```

**`add_route`** — accept `mode: "http"` (default) or `mode: "tcp"`
- HTTP mode: same as today (requires `domain:`)
- TCP mode: call `allocate_tcp_port` inside a transaction, create route without domain

**New private `allocate_tcp_port`**
```ruby
def allocate_tcp_port
  used = Route.where(mode: "tcp").lock.pluck(:public_port).to_set
  TCP_PORT_RANGE.find { |p| !used.include?(p) } ||
    raise Error, "No TCP ports available (pool 3000–3099 exhausted)"
end
```
Wrap the create + allocate in `Route.transaction` to prevent races.

**`write_config`** — split into http and tcp sections in the same `sandbox-{id}.yml`:
```yaml
http:
  routers:
    sandbox-5-r1:
      rule: "Host(`app.example.com`)"
      entryPoints: [websecure]
      ...
  services: ...
tcp:
  routers:
    sandbox-5-r2:
      rule: "HostSNI(`*`)"
      entryPoints: [tcp-3042]
      service: sandbox-5-r2
  services:
    sandbox-5-r2:
      loadBalancer:
        servers:
          - address: "alice-mybox:8080"
```

TCP router key: `sandbox-{sandbox.id}-tcp-r{route.id}`
HTTP router key: `sandbox-{sandbox.id}-r{route.id}` (unchanged)

## Traefik static config — on-demand entrypoints

Traefik entrypoints are static (require restart to add). Rather than pre-declaring the full range,
entrypoints are added **on demand**: when a TCP route is created for a port not yet in `traefik.yml`,
the app adds the entrypoint and restarts the Traefik container (~1 s downtime).

**`RouteManager#ensure_tcp_entrypoint(port)`** (new private method):
1. Read `TRAEFIK_STATIC_CONFIG` path (e.g. `/data/traefik/traefik.yml`)
2. Parse YAML; check if entrypoint `tcp-{port}` already exists
3. If missing: add it, write the file back, then `Docker::Container.get("sandcastle-traefik").restart`
4. If already present: no-op

**`add_route` for tcp mode calls `ensure_tcp_entrypoint(public_port)` before writing dynamic config.**

The `SANDCASTLE_TRAEFIK_CONFIG` env var (default `/data/traefik/traefik.yml`) tells Rails where the
static config lives. The Traefik container name is `sandcastle-traefik` (configurable via
`SANDCASTLE_TRAEFIK_CONTAINER`).
