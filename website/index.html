Get Started How It Works

Designed for small teams of 2–10 people sharing a dedicated root server — like a Hetzner dedicated box, OVH bare-metal, or any Linux host with Docker.

Everything Your Team Needs

Full isolation, full power. Each sandbox is a real Linux environment with root access and its own Docker daemon.

True Container Isolation

Each sandbox runs in a Sysbox container — real isolation with its own PID/network/mount namespaces. No stepping on each other's toes.

Docker Inside Docker

Every sandbox includes a working Docker daemon. Build images, run compose stacks, test deployments — all inside your sandbox.

SSH Access

Connect via SSH with your own keys. Use VS Code Remote, JetBrains Gateway, or plain terminal. Each sandbox gets a unique port.

Persistent Storage

User home directories and workspace volumes survive container restarts. Your dotfiles, repos, and data stay put.

Web Dashboard

A clean Rails-powered admin UI to create, start, stop, and monitor sandboxes. See container stats and manage users at a glance.

CLI Tool

A fast Go CLI to manage everything from your terminal. Create sandboxes, SSH in, snapshot and restore — all in one command.

Tailscale Networking

Optional per-user Tailscale sidecar for private mesh networking. Access your sandbox from anywhere without exposing ports.

Snapshot & Restore

Save the state of any sandbox as a snapshot. Roll back to a known-good state or clone an environment in seconds.

Fully Self-Hosted

No cloud dependency, no SaaS fees. Run on your own hardware. Your code stays on your server, period.

Why Sandcastle?

Cloud dev environments are great — until you see the bill. If your team already has a powerful dedicated server (or wants to rent one), Sandcastle turns it into a shared development platform in minutes.

Fraction of the cost

A Hetzner AX102 (128 GB RAM, 2x 1TB NVMe) costs ~$130/mo. That's $13/person for a 10-person team with way more power than any cloud dev environment.

Full control

You own the server, you own the data. No vendor lock-in, no telemetry, no surprises. Customize the sandbox image to your exact needs.

Real Linux, real Docker

Not a stripped-down container or a browser IDE. You get a full Ubuntu environment with root, systemd, and a real Docker daemon. Use any editor, any tool.

Built for trust

Sandcastle assumes users are trusted teammates, not adversaries. This keeps the system simple and the overhead low. Perfect for co-founders, small startups, and dev collectives.

Up and Running in 5 Minutes

One bootstrap script installs everything. Three commands later, your team has sandboxes.

1

Bootstrap Your Server

SSH into your fresh Linux server and run the bootstrap script. It installs Docker, Sysbox, Caddy (for HTTPS), and configures the firewall.

root@server:~# curl -sSL https://get.sandcastle.rocks | bash
2

Deploy Sandcastle

Use Docker Compose or Kamal to deploy the Rails app. Set your domain and secret key, and you're live.

root@server:~# docker compose up -d
sandcastle-web | Listening on 0.0.0.0:3000
sandcastle-jobs | SolidQueue started
3

Create Sandboxes

Use the web UI or CLI to create sandboxes for your team. Each person gets their own isolated environment with SSH access, a Docker daemon, and persistent storage.

$ sandcastle create --name alice
$ sandcastle create --name bob
$ sandcastle list
NAME STATUS SSH PORT DOCKER
alice running 2201 ready
bob running 2202 ready

Done! Start building.

Your teammates can now SSH in, spin up Docker containers, and work in their own isolated environments. All on one server, for a fraction of the cloud cost.
