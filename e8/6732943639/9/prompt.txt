2. Key Technical Concepts:
   - **RFB protocol**: Server speaks first (sends `RFB 003.008\n` banner immediately on connection)
   - **x11vnc 0.9.17 WebSocket detection**: New behavior reads client data first before sending RFB banner — creates deadlock with websockify (which also waits for server to speak first)
   - **TigerVNC Xvnc**: Combined X display + VNC server in a single process; sends RFB banner immediately; flags: `-rfbport 5900 -SecurityTypes None -AlwaysShared -geometry 1920x1080 -depth 24`
   - **websockify**: WebSocket→raw TCP proxy; converts browser WebSocket frames to raw RFB for x11vnc/Xvnc
   - **gotget/novnc image**: Old 2019 image with bundled websockify that was not converting WS→TCP properly (sending WebSocket handshake to x11vnc)
   - **Traefik stripPrefix**: `/vnc/{id}` (not `/vnc/{id}/novnc`) so noVNC asset relative paths resolve correctly
   - **Sysbox runtime**: Does NOT support Docker's `ShmSize` HostConfig key — causes "permission denied" on entrypoint; workaround is `mount -o remount,size=2g /dev/shm` inside container
   - **Sysbox isolated kernel**: Each sandbox container gets its own kernel view, so remounting tmpfs works
   - **VncController forwardAuth**: Traefik calls `/vnc/auth` with `X-Forwarded-Uri`; regex must match current URL scheme
   - **noVNC sidecar architecture**: Runs websockify + serves static files; connects to `sandcastle-web` network; routes to `sc-vnc-{name}:6080`
   - **WeTTY sidecar architecture**: Node.js SSH→WebSocket bridge; ephemeral ed25519 keypair injected via `docker exec`; routes to `sc-wetty-{name}:3000`
   - **ttyd**: Lightweight C binary alternative to WeTTY; runs shell command over WebSocket; no Node.js required
   - **pgdata Docker named volume**: Stores postgres data; destroyed by `docker compose down --volumes`
